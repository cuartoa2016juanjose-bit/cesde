##  Programación Orientada a Objetos (POO)
El núcleo de la lógica empresarial. Implementación de patrones de diseño y principios de código limpio para crear software altamente modular.

| Concepto | Aplicación en el Proyecto |
| :--- | :--- |
| **Clases y Objetos** | Creación de moldes precisos y abstracción de entidades del mundo real hacia el modelo de negocio. |
| **Encapsulamiento** | Protección de la integridad de los datos mediante modificadores de acceso (`private`, `protected`) y métodos mutadores/accesores. |
| **Herencia** | Estructuración jerárquica para la máxima reutilización de código (DRY - *Don't Repeat Yourself*). |
| **Polimorfismo** | Diseño de sistemas dinámicos donde un mismo método puede comportarse de múltiples formas según el contexto de ejecución. |
| **Interfaces** | Definición de contratos estrictos para garantizar un comportamiento estandarizado entre clases no relacionadas. |
| **Organización por Paquetes** | Arquitectura limpia estructurada en `models`, `controllers`, `views` y `services` para facilitar el mantenimiento. |

##  Dominación del Backend ("DOOM Mode")
Control absoluto sobre el servidor, la lógica de negocio y la comunicación de datos.

* **Arquitectura de Software:** Separación de responsabilidades y estructuración lógica del servidor.
* **Flujo de Datos:** Procesamiento eficiente de peticiones, validación de entradas y sanitización de datos.
* **Integración:** Conexión fluida entre la lógica de la POO y las capas de persistencia (Bases de Datos).
* **Escalabilidad:** Código diseñado no solo para funcionar hoy, sino para soportar el crecimiento de módulos mañana sin colapsar.

##  Metodologías Ágiles y Scrum
El código excelente requiere una gestión excelente. Aplicación de metodologías de la industria para entregas continuas y de alto valor.

* **Ciclo de Vida Iterativo:** Desarrollo basado en **Sprints** cortos para adaptación rápida a los cambios.
* **Artefactos Scrum:** * Gestión y priorización del **Product Backlog**.
  * Definición de tareas accionables en el **Sprint Backlog**.
* **Ceremonias Ágiles:**
  *  *Daily Stand-ups:* Sincronización del equipo y bloqueo de impedimentos.
  *  *Sprint Planning:* Estimación de esfuerzo y definición de objetivos claros.
  *  *Sprint Review & Retrospective:* Demostración de valor y mejora continua del proceso de desarrollo.
